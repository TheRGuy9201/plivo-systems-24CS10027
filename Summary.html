Reliable UDP Streaming System

Objective

The objective of this project is to improve reliability of real-time UDP media streaming over a hostile network while satisfying strict playback deadlines.

Sender Design

  • Receives frames from the harness.
  • Creates a custom packet format.
  • Sends every frame as a DATA packet.
  • Groups frames in pairs.
  • Generates one XOR parity packet for each group.

Receiver Design

  • Parses DATA and PARITY packets.
  • Stores packets using group-based buffering.
  • Uses XOR parity to recover one missing frame.
  • Forwards both original and reconstructed frames to the player.

Packet Format

+---------+--------------+--------------+
| Type    | Seq/Group ID | Payload      |
+---------+--------------+--------------+

Type = DATA or PARITY.

Forward Error Correction

For every two frames, one parity packet is generated: 

Parity = Frame0 XOR Frame1

If exactly one frame is lost, it can be reconstructed using: 

Missing = Parity XOR Remaining Frame

Current Limitations

  • No packet reordering.
  • No jitter buffer.
  • No retransmission.
  • Cannot recover multiple losses within a group.

Future Work

  • Packet reordering.
  • Adaptive playback buffer.
  • ACK/NACK-based retransmission.
  • Adaptive FEC group sizing.
  • Duplicate suppression.
